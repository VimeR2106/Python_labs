## Условия задач

### Сложность: Rare
Решите обе задачи своего варианта:
1. Замыкание для получения текста ответа на запрос к API (например, `https://dogapi.dog/api/v2/facts`).
2. Декоратор, ограничивающий частоту вызовов функций.

Дополнительно:
- Применить декоратор к замыканию.

### Сложность: Medium
- Создать декоратор с опциональным параметром.
- Продумать поддержку рекурсивных функций.

### Сложность: Well-done
- Реализовать декоратор классов вместо декоратора функций.

## Описание проделанной работы

- В файле `lab4_closures.py` реализован декоратор-класс `RateLimiter`.
- Декоратор поддерживает два варианта использования:
  - `@RateLimiter`
  - `@RateLimiter(calls=2, period=1.0)`
- Для рекурсивных функций добавлена корректная обработка: ограничение применяется к верхнеуровневому вызову, а внутренние рекурсивные вызовы не «штрафуются» отдельно.
- Реализовано замыкание `make_api_text_fetcher(...)`, которое:
  - хранит URL API во внешней области видимости;
  - запрашивает данные из API `dogapi.dog`;
  - возвращает текст факта о собаках;
  - декорируется `RateLimiter`, то есть к замыканию применён декоратор ограничения частоты.
- Добавлена функция `demo()` для демонстрации работы всех требований задания.
