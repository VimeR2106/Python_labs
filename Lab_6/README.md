## Условия задач

Реализовать расчёт параметров геометрических тел (параллелепипед, тетраэдр, шар):

## Описание проделанной работы

### Пакет `geometry`

Пакет содержит три модуля, каждый реализует три функции: `volume()`, `surface()`, `mass()`.

**`parallelepiped.py`** — принимает стороны `a, b, c`:
$$V = a \cdot b \cdot c \qquad S = 2(ab + bc + ac)$$

**`tetrahedron.py`** — принимает ребро `a`:
$$V = \frac{a^3}{6\sqrt{2}} \qquad S = \sqrt{3}\,a^2$$

**`sphere.py`** — принимает радиус `r` (передаётся как параметр `a`):
$$V = \frac{4}{3}\pi r^3 \qquad S = 4\pi r^2$$

Масса во всех случаях: $m = \rho \cdot V$

### Графический интерфейс (Kivy)

`main.py` строит интерфейс из стандартных виджетов Kivy (размер окна 480×640):
- `Spinner` — выбор фигуры (`Параллелепипед`, `Тетраэдр`, `Шар`) и материала
- `TextInput` — три поля ввода `a`, `b`, `c`; для тетраэдра и шара используется только `a`
- Подсказка под спиннером фигуры меняется при смене фигуры
- Валидация: перед расчётом проверяется, что все введённые значения являются числами
- `Button «Рассчитать»` — вычисляет объём, площадь и массу, выводит результат
- `Button «Сохранить в Excel»` — доступна всегда
- `Button «Сохранить в БД (PostgreSQL)»` — появляется **только** если при запуске удалось импортировать `database.py` (т.е. БД доступна)

### Сложность Rare — сохранение в Excel

`exporter.py` использует `openpyxl==3.1.2`. При первом сохранении создаётся файл `results.xlsx` с заголовками; каждый следующий результат добавляется новой строкой. Значения объёма, площади и массы сохраняются **округлёнными** до целых.

### Сложность Medium — PostgreSQL в Docker

`database.py` использует `psycopg[binary]==3.2.13` (psycopg3). Параметры подключения берутся из переменных окружения (с дефолтными значениями):

| Переменная    | По умолчанию  |
|---------------|---------------|
| `DB_HOST`     | `localhost`   |
| `DB_PORT`     | `5433`        |
| `DB_NAME`     | `geometry_db` |
| `DB_USER`     | `user`        |
| `DB_PASSWORD` | `password`    |

Функция `init_db()` создаёт таблицу `results` (если не существует). Функция `save_to_db()` добавляет строку. `init_db()` вызывается автоматически при старте приложения.

## Запуск

```bash
# 0. Создать виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate

# 1. Установить зависимости
pip install -r requirements.txt

# 2. (Medium) Запустить PostgreSQL в Docker
docker compose up -d

# 3. Запустить приложение
python main.py
```

> Если Docker недоступен, приложение запустится без кнопки «Сохранить в БД».
